# Qunyan Training Prompt vs Skill Split

這份文件定義群雁培訓系統中，哪些能力應該做成：

- `Prompt`
- `Skill / 流程`
- `混合型（Skill + Prompt）`

目的是避免後續開發時把本來應該固定的流程交給 prompt，或把本來只需要調文案的地方硬寫死在程式裡。

相關文件：

- `docs/qunyan_training_system_plan.md`
- `docs/qunyan_training_phase1_data_model.md`

## 一、判斷原則

## 適合做 Prompt 的情況

如果主要差異是：

- 語氣
- 結構
- 文案風格
- 教學說明方式
- 總結寫法

那就應該放在 `Prompt`。

## 適合做 Skill / 流程 的情況

如果主要差異是：

- 多步驟操作
- 需要讀取多個資料來源
- 狀態轉換
- 驗證與限制
- 需要穩定重複執行

那就應該做成 `Skill / 流程`。

## 適合做混合型的情況

如果前段需要固定流程判斷，最後輸出又需要 AI 幫忙整理語氣與文字品質，就做成：

- `Skill / 流程` 負責前段
- `Prompt` 負責最後輸出

## 二、群雁培訓系統功能分類

## 1. 培訓模組中心

### 建議類型

- `Skill / 流程`

### 原因

這是結構化資料管理，不是生成文案。

### 包含

- 新增模組
- 查詢模組
- 更新模組
- 掛教材附件

## 2. 培訓課程/場次管理

### 建議類型

- `Skill / 流程`

### 原因

涉及資料建立、查詢、排序、狀態管理，不能只靠 prompt。

### 包含

- 新增培訓課程
- 查詢培訓課程
- 修改培訓課程
- 與行事曆整合

## 3. 悟到 / 學到 / 做到 / 目標 填寫

### 建議類型

- `Skill / 流程`

### 原因

本質是固定欄位輸入與儲存。

### 包含

- 新增課後反思
- 查詢歷史反思
- 查詢某人反思紀錄

## 4. 課後反思整理摘要

### 建議類型

- `Prompt`

### 原因

資料本身已經固定，AI 的價值在於幫忙整理成更好讀的摘要。

### 可做成 Prompt 的項目

- 把多位夥伴的 `悟到` 彙整成共通體悟
- 把 `做到` 統整成團隊本週行動方向
- 把 `目標` 彙整成團隊目標看板文案

## 5. 七天法則追蹤

### 建議類型

- `Skill / 流程`

### 原因

它是 onboarding checklist，不是單純文案。

### 包含

- 開立七天任務
- 每日完成狀態
- 缺失提醒
- 上線追蹤

## 6. 七天法則提醒與鼓勵文案

### 建議類型

- `Prompt`

### 原因

提醒內容、鼓勵話術、打預防針的說法，適合交給 prompt 管理。

### 可做成 Prompt 的項目

- 新人第 1 天鼓勵訊息
- 新人第 3 天預防針提醒
- 新人第 7 天總結文案

## 7. 2689 帶線追蹤

### 建議類型

- `Skill / 流程`

### 原因

這是結構化追蹤系統，要記錄月份、線、深度、寬線與日誌。

### 包含

- 2 條線
- 6 個月帶線
- 8 代市場
- 9 條寬線
- 有效系統 / 報表系統判定

## 8. 2689 教學說明與帶線建議

### 建議類型

- `混合型`

### Skill 部分

- 判斷目前缺的是 2 / 6 / 8 / 9 哪一段
- 根據資料找出哪條線停滯
- 判斷有效系統或報表系統

### Prompt 部分

- 生成給上線看的建議說明
- 生成對夥伴的鼓勵與提醒話術

## 9. 市場實戰案例庫

### 建議類型

- `Skill / 流程`

### 原因

案例要用固定欄位存：

- 情境
- 動作
- 結果
- 關鍵

### 包含

- 新增案例
- 查詢案例
- 搜尋案例
- 依產品/地區/情境分類

## 10. 案例轉成分享稿

### 建議類型

- `Prompt`

### 原因

這一步重點是文字表達，而不是資料結構。

### 可做成 Prompt 的項目

- 案例簡報文案
- 小組分享稿
- 對外故事版本

## 11. 培訓進度追蹤

### 建議類型

- `Skill / 流程`

### 原因

是資料累積、統計與推薦，不是單純生成內容。

### 包含

- 完成模組
- 完成場次
- 最近反思
- 下個推薦模組

## 12. 培訓進度 AI 建議

### 建議類型

- `混合型`

### Skill 部分

- 根據完成模組與反思內容判斷目前階段
- 找出落後點與下一步

### Prompt 部分

- 寫出給夥伴看的建議內容
- 寫出給帶領人的跟進摘要

## 13. 領導人能力評估

### 建議類型

- `混合型`

### Skill 部分

- 收集四個勇於的評估分數
- 做月份趨勢
- 判斷低分區塊

### Prompt 部分

- 生成成長建議
- 生成回饋文字

## 三、第一期最適合先做的 Prompt

第一期若只先配少量 prompt，最值得先做的是：

### 1. `training_reflection_summary`

用途：

- 將一堂課的多位反思整理成群體摘要

### 2. `training_action_suggestion`

用途：

- 根據某人的悟到/學到/做到/目標，生成下一週行動建議

### 3. `training_progress_feedback`

用途：

- 根據培訓進度給個人回饋

## 四、第一期最適合先做的 Skill

### 1. `training-module-management`

用途：

- 模組新增、查詢、更新、掛教材

### 2. `training-session-management`

用途：

- 課程場次建立、查詢、同步行事曆

### 3. `training-reflection-workflow`

用途：

- 填寫悟到/學到/做到/目標
- 儲存
- 查詢

### 4. `training-progress-tracker`

用途：

- 依課程與反思更新個人進度
- 推出個人/團隊進度摘要

## 五、建議的實作順序

## 第一階段

先做 Skill / 流程：

- training-module-management
- training-session-management
- training-reflection-workflow
- training-progress-tracker

## 第二階段

再補 Prompt：

- training_reflection_summary
- training_action_suggestion
- training_progress_feedback

## 第三階段

再做混合型：

- 七天法則提醒
- 2689 帶線建議
- 領導人能力評估建議

## 六、總結

群雁培訓系統最適合的分工是：

- `Skill / 流程`
  - 負責資料、步驟、狀態、查詢、驗證

- `Prompt`
  - 負責摘要、回饋、鼓勵、建議文案

- `混合型`
  - 先用 skill 判斷情境，再用 prompt 生成文字

這樣系統才會同時具備：

- 穩定性
- 可維護性
- 可調整性
- 後續 AI 優化空間
