# LINE 待歸檔兩階段測試清單

適用功能：
- 上傳圖片 / 音檔 / 影片 / 檔案後，先進 `output/pending_archive`
- LINE 回覆數字後，再執行實際歸檔
- 支援人員 + 模式組合、一般模式、行事曆特殊選項

測試前準備：
- 啟動 webhook

```powershell
python agents/06_line_webhook.py
```

- 確認 LINE Bot webhook 可正常收到訊息
- 準備 3 份素材
  - 1 張一般圖片
  - 1 份 PDF 或 TXT 檔
  - 1 張行事曆截圖

## 1. 圖片待歸檔選單

步驟：
1. 上傳 1 張一般圖片
2. 觀察 LINE 回覆

預期結果：
- 不直接歸檔到正式分類目錄
- 回傳待歸檔選單
- 選單至少包含：
  - `1. 宜芸 整理會議心得`
  - `7. 建德 歸檔行動紀錄`
  - `13. 行事曆圖檔`
  - `14. 行事曆文字新增`

可檢查路徑：
- `output/pending_archive/files`
- `output/pending_archive/states`

## 2. 數字執行歸檔

步驟：
1. 先上傳 1 份圖片或檔案
2. LINE 回覆 `7`

預期結果：
- LINE 回覆：
  - `已執行：建德 歸檔行動紀錄`
- 待歸檔狀態被清除
- 檔案進入：
  - `output/classified/建德/歸檔行動紀錄/YYYYMMDD/...`

## 3. 重新查看待歸檔

步驟：
1. 上傳任一檔案
2. LINE 輸入：
  - `待歸檔`
  - 或 `查詢待歸檔`

預期結果：
- 系統重新回傳同一份待歸檔數字選單

## 4. 一般模式歸檔

步驟：
1. 上傳 1 份檔案
2. LINE 回覆 `12`

預期結果：
- LINE 回覆：
  - `已執行：歸檔文件`
- 檔案進入：
  - `output/classified/歸檔文件/YYYYMMDD/files`

## 5. 行事曆圖檔

步驟：
1. 上傳 1 張行事曆截圖
2. LINE 回覆 `13`

預期結果：
- 走行事曆圖片整理流程
- 回覆今天之後的行事曆整理內容
- 行事曆資料更新到：
  - `output/calendar/calendar_events.json`

## 6. 行事曆文字新增

步驟：
1. 上傳任一附件
2. LINE 回覆 `14`
3. 接著輸入：

```text
小幫手 新增行事曆 2026-04-05 19:30 團隊會議 | 台北教室
```

預期結果：
- 第一步會先回覆：
  - 附件已存入行事曆手動新增目錄
  - 並提示後續輸入格式
- 第二步會成功新增行事曆

可檢查路徑：
- `output/classified/行事曆/YYYYMMDD/manual_add`

## 7. 無待歸檔時輸入數字

步驟：
1. 在沒有任何待歸檔項目的情況下
2. 直接輸入 `7`

預期結果：
- 不應執行任何歸檔
- 會走原本文字意圖流程，或無動作

## 8. 歸類模式並存驗證

步驟：
1. 輸入：

```text
小幫手 歸類模式 建德 會議記錄
```

2. 上傳 1 張圖片
3. LINE 回覆 `6`

預期結果：
- 即使已設定歸類模式，附件仍先進待歸檔
- 回覆數字後才正式執行
- `6` 應執行：
  - `建德 歸檔會議紀錄`

## 9. 回歸測試

步驟：
1. `小幫手 查詢行事曆`
2. `小幫手 查詢夥伴 吳建德`
3. `小幫手 查詢歸檔`

預期結果：
- 三個既有功能仍可正常回覆

## 建議驗收順序

1. 圖片待歸檔選單
2. 回覆 `7`
3. 回覆 `待歸檔`
4. 回覆 `13`
5. 回覆 `14` 後再執行 `新增行事曆`
6. 查詢行事曆 / 查詢歸檔 / 查詢夥伴

## 備註

- PowerShell 若顯示 emoji 變成 `?`，通常只是終端編碼問題，不影響 LINE 實際訊息。
- 若測試時發現選單人員不足，代表 `output/classified` 內尚未有對應人員資料夾；目前選單來源是現有歸檔人員。
